Load Balancer & Auto-Scaling Simulation

System Overview

1. Initialization

The program starts with an initialization where the user inputs their custom parameters. Then, the system starts the simulation, creating web servers and requests to be fulfilled.

2. Arrivals

For each cycle, a random number of requests arrives. While there are still requests to be fulfilled, they are checked to see if their IP is blocked, then added to the queue if not blocked.

3. Queue

If the queue is not empty, and an idle server is available, the first request in the queue is assigned to a server. If an idle server isn't available, the request remains in the queue.

4. Servers

Each server is checked to see if the request has finished. If so, the request is removed from the server, and statistics are updated. If the request isn't finished, it remains on the server.

5. Scaling

The system changes the number of servers based on the number of tasks. If Queue > 80 * servers, servers are added. if Queue < 50 * servers and more than one server exists, a server is removed.

6. Reporting

Performance metrics such as total processed requests, total servers utilized, and duration of time are reported.
